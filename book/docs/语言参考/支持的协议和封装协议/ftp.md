# ftp://

允许通过 FTP 读取存在的文件，以及创建新文件。如果服务器不支持被动模式的 FTP ，连接会失败。打开文件后既可以读也可以写，但是不能同时进行。当远程文件已经存在于 FTP 服务器上，如果尝试打开并写入文件的时候，未指定上下文选项 `overwrite` ，连接会失败，如果要通过 FTP 覆盖存在的文件，需要指定上下文的 `overwrite` 选项。

|           属性            | 支持 |
| :-----------------------: | :--: |
| 受 `allow_url_fopen` 限制 | Yes  |
|         允许读取          | Yes  |
|         允许写入          | Yes  |
|         允许添加          | Yes  |
|      允许同时读和写       |  No  |
|       支持 `stat()`       | Yes  |
|      支持 `unlink()`      | Yes  |
|      支持 `rename()`      | Yes  |
|      支持 `mkdir()`       | Yes  |
|      支持 `rmdir()`       | Yes  |
